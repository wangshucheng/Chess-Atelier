# 2026-07-20

## 修复：棋盘显示异常 + getBoundingClientRect 报错

### 关键结论
- 项目使用 **HashRouter**（`src/App.tsx`），正确页面 URL 形如 `http://localhost:5173/chess/#/play`。
  之前用 `/chess/play`（path 形式）会被当成空 hash → 匹配 `/` → 渲染**首页（无棋盘）**，这是“棋盘显示异常”的直接原因。
- react-chessboard 4.7.3 本身正常（浏览器内隔离渲染 64 格 + 32 棋子成功）。

### 报错根因
react-chessboard 在 `boardWidth` 缺省时依赖内部 ResizeObserver 测量容器宽度，
该测量会在 null 引用上调用 `getBoundingClientRect`，导致 `Uncaught TypeError: Cannot read properties of null (reading 'getBoundingClientRect')`。

### 修复内容
1. `src/components/board/ChessBoard.tsx`：自行用 `useLayoutEffect` + ResizeObserver 测量父容器宽度，
   显式 `boardWidth={resolvedWidth}`（未测到时用兜底 480），不再依赖库内置测量路径。
2. `src/main.tsx`：移除了 `React.StrictMode`（react-chessboard 4.7.x 在 StrictMode 双挂载下已知有棋盘不渲染缺陷）。
   注：主要修复是显式 boardWidth；StrictMode 移除作为双保险。

### 验证
- `/#/play`：squares=64, 棋子=32 ✅
- `/#/puzzles/1`：squares=64（残局棋子少属正常）✅
- Review/Puzzles 列表页初始 squares=0 是正常的（需导入/异步加载棋局后才渲染棋盘）。

### 注意事项
- 访问页面必须用 HashRouter 形式（带 `#`），否则会落到首页。
- Playwright MCP 的 `evaluate` 返回值不可靠（常返回空 `{}`/`3`），需用 DOM 回写或 `querySelectorAll` 计数方式验证。

## 部署：GitHub Pages（gh-pages）

### 部署目标
- 远程仓库 `wangshucheng/Chess-Atelier`（GitHub）。
- 部署方式：`npm run deploy` = `tsc -b && vite build` + `gh-pages -d dist`（推送 dist 到远程 `gh-pages` 分支，gh-pages 默认会写 `.nojekyll`）。

### 本次修正（否则 Pages 上资源 404）
1. `vite.config.ts`：`base` 从 `/chess/`（注释误写仓库名 chess）改为 `/Chess-Atelier/`，与实际仓库名一致。
2. `index.html`：favicon 硬编码 `/chess/favicon.svg` → `/favicon.svg`，让 Vite 构建时自动注入 base。
   - 验证：构建后 `dist/index.html` 中所有资源均为 `/Chess-Atelier/...`，favicon 已正确变为 `/Chess-Atelier/favicon.svg`。

### 当前阻塞
- 当前 IDE 环境**无外网、无代理**，`gh-pages` 克隆远程时报 `Failed to connect to github.com port 443 ... Timed out`（已重试两次）。
- 解决二选一：
  a. 在能联网的本机进入项目执行 `npm run deploy`；
  b. 若本环境需代理，先 `git config --global http.proxy <地址:端口>` 后再 `npm run deploy`。

- 仓库 Settings → Pages → Source 选 `Deploy from a branch` → Branch 选 `gh-pages` / `(root)` → Save。
- 访问地址：`https://wangshucheng.github.io/Chess-Atelier/`
- 路由为 HashRouter，子页形如 `/#/play`、`/#/puzzles/1`（base 已是 /Chess-Atelier/）。
- 注意：公开仓库 Pages 免费；私有仓库需付费计划。仓库名大小写（Chess-Atelier）需与 base 一致，否则资源 404。
